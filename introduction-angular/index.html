Frontend development

  • CSS adds presentational properties for this composition, aesthetically pleasing effects and display responsiveness
  • JavaScript is the programming language of the web. It adds the possibility of interaction and dynamic behavior

Frontend frameworks

  • Promise increased developer productivity, by providing solutions to common and often hard challenges
  • They are the backbone of modern Single Page Applications: highly dynamic, data-driven apps that act and feel like a native app
  • Common challenges include event handling, data modelling, DOM manipulation, async operations, routing, state management, …

Angular

  • Low decision fatigue, less freedom, enforced consistency between apps and developers
  • Excellent tooling and testability: TypeScript and Angular CLI
  • Hard to learn, Angular-specific tags and mandatory tooling

React

  • The 'view' layer of your app
  • Requires further choices of libraries for common functionality
  • JSX, put HTML in JS, fail fast and explicit

”Choosing between Angular and React is like choosing between buying an off-the-shelf computer and building your own with off-the-shelf parts”

Web components

HTML is our primary means of describing the GUI, but it is (currently) not extensible or very expressive.

Extend HTML with reusable custom element tags that encapsulate style, markup and functionality to build rich applications.

Components

  • Web components drive Separation of Concerns, enable functional re-use and haven give rise to the development of style guides
  • Components are isolated, have clear I/O, are declarative and composable, making them easy to reason about

Angular is what HTML would have been, if it was used to develop dynamic web applications.

Angular enables you to create reusable components (custom HTML selectors) and combine them with services to form a web app.

A component is the combination of a HTML template with related styling and a component class that controls a portion of the screen.


import { Component } from '@angular/core';

@Component({
  selector: 'my-app',
  template: `<h1>Hello {{name}}</h1>`
})
export class AppComponent {
  name = 'Angular';
}

Every component begins with an @Component decorator function that takes a metadata object. The metadata object describes how the HTML template and component class work together.


import { Component } from '@angular/core';

The selector property tells Angular to display the component inside a custom <my-app> tag in the index.html.


import { Component } from '@angular/core';

@Component({
  selector: 'my-app',
  template: `<h1>Hello {{name}}</h1>`
})
export class AppComponent {
  name = 'Angular';
}

The template property defines a message inside a header. The message starts with "Hello" and ends with {{name}}, which is an Angular interpolation binding expression. At runtime, Angular replaces {{name}} with the value of the component's name property.


import { NgModule }      from '@angular/core';
import { BrowserModule } from '@angular/platform-browser';
import { AppComponent }  from './app.component';

@NgModule({
  imports:      [ BrowserModule ],
  declarations: [ AppComponent ],
  bootstrap:    [ AppComponent ]
})
export class AppModule { }

The module definition describes all the parts of the application and which root component should be loaded. 


import { platformBrowserDynamic }
			from '@angular/platform-browser-dynamic';
import { AppModule } 	from './app/app.module';

platformBrowserDynamic().bootstrapModule(AppModule);

The bootstrap function kicks off Angular with our module after the page is loaded.


import { Component } from '@angular/core';

@Component({
  selector: 'avatar',
  template: `<img [src]="imageUrl" />`
})
export class AvatarComponent {
  imageUrl = 'https://example.com/images/mr-awesome.png';
}

You can bind to properties using square brackets, such as the image src property above.


@Component({
  selector: 'shoutout-button',
  template: `
    <button (click)="onClick()">Say what?!</button>
    {{message}}
  `
})
export class ShoutoutComponent {
  message = '';

  onClick() {
    this.message = 'Hello, Angular meetup!!';
  }
}

You can bind to events using parentheses, such as the button click event above. 


import {Injectable} from '@angular/core';
import {Http} from '@angular/http';

@Injectable()
export class ApiService {
  constructor(private http: Http) { }

  getUserData(username: string) {
    return this.http.get(`/api/users/${username}`)
      .map(result => result.json());
  }
}

Services are simple TypeScript classes that offer reusable functionality.


@Component({
  selector: 'user',
  template: `
    <input type="text" #username />
    <button (click)="getUser(username.value)" />
    Hello, {{user.name}}!
  `
})
export class UserComponent {
  user = {};

  constructor(private api: ApiService) {}

  getUser(username) {
    this.user = this.api.getUserData(username);
  }
}

Dependency injection is used to grab hold of the services you want to use.
